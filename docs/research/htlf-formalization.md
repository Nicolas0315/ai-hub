## 1. 対応関係の形式的表現

### 1.1 二部グラフモデル

翻訳元レイヤー S（source）と翻訳先レイヤー T（target）の要素集合を考える。

- **S = {s₁, s₂, …, sₘ}** — ソース要素（例: 論文の数式・定義・主張）
- **T = {t₁, t₂, …, tₙ}** — ターゲット要素（例: ニュース記事の文・図・見出し）

**対応関係** は二部グラフ G = (S ∪ T, E) として表現する:

```
E ⊆ S × T
(sᵢ, tⱼ) ∈ E  ⟺  sᵢ が tⱼ に翻訳（部分的にでも）対応している
```

各辺に**対応強度** w: E → [0, 1] を付与する（完全一致=1、部分的=0〜1）。

### 1.2 要素の分類

対応関係 G が与えられたとき、各要素は以下の3カテゴリに分類される:

| カテゴリ | ソース側 | ターゲット側 |
|---------|---------|------------|
| **Matched** | S_M = {s ∈ S : ∃t, (s,t) ∈ E} | T_M = {t ∈ T : ∃s, (s,t) ∈ E} |
| **Unmatched (Lost)** | S_U = S \ S_M | — |
| **Unmatched (Added)** | — | T_U = T \ T_M |

- **S_U**: ソースにあるがターゲットに対応がない要素 → **翻訳損失の候補**
- **T_U**: ターゲットにあるがソースに対応がない要素 → **翻訳時の付加情報**（誤情報の候補でもある）

## 2. 損失量の逆算

### 2.1 基本損失関数

マッチした要素ペア (sᵢ, tⱼ) ∈ E に対し、意味的距離関数 d を定義:

```
d: S × T → [0, 1]
d(sᵢ, tⱼ) = 0  ⟺  完全な意味保存
d(sᵢ, tⱼ) = 1  ⟺  意味的に無関係
```

**要素レベル損失:**

```
ℓ(sᵢ, tⱼ) = d(sᵢ, tⱼ)        （マッチした要素の部分損失）
ℓ(sᵢ)     = 1                  （マッチしなかった要素の完全損失）
```

**集約損失（ソース要素の重要度 π で加重）:**

```
L_raw = Σᵢ πᵢ · ℓᵢ / Σᵢ πᵢ
```

ここで πᵢ はソース要素 sᵢ の重要度（Σπᵢ = 1 に正規化可能）。
ℓᵢ = min_{(sᵢ,tⱼ)∈E} d(sᵢ, tⱼ) （マッチあり）、ℓᵢ = 1 （マッチなし）。

### 2.2 復元度

```
R = 1 - L_raw ∈ [0, 1]
```

## 3. 三軸への分解

意味的距離 d を3つの独立した軸に分解する。各軸は異なる意味次元の距離を測る。

### 3.1 軸別距離関数

```
d_struct(s, t)  : 構造的関係（論理構造・演算・関係性）の距離
d_context(s, t) : 文脈的意味（前提知識・暗黙の仮定・分野固有の背景）の距離
d_qualia(s, t)  : 質感的意味（体験的・感覚的・美的特質）の距離
```

**操作的定義:**

| 軸 | 測定対象 | 操作的指標例 |
|----|---------|------------|
| d_struct | 論理関係・因果構造・数式構造の保存 | 命題論理への還元後の同型度、依存グラフの編集距離 |
| d_context | 前提条件・スコープ・適用範囲の保存 | 前提条件の再現率、限定子（"in certain conditions"等）の保持率 |
| d_qualia | 感覚的印象・感情的反応の保存 | 感情極性の一致度、強度語の対応（"breakthrough" vs 原文のトーン） |

### 3.2 軸別復元度

各軸について §2 と同じ集約を適用:

```
R_struct  = 1 - Σᵢ πᵢ · ℓᵢ_struct  / Σᵢ πᵢ
R_context = 1 - Σᵢ πᵢ · ℓᵢ_context / Σᵢ πᵢ
R_qualia  = 1 - Σᵢ πᵢ · ℓᵢ_qualia  / Σᵢ πᵢ
```

ここで ℓᵢ_axis = min_{(sᵢ,tⱼ)∈E} d_axis(sᵢ, tⱼ) （マッチあり）、ℓᵢ_axis = 1 （マッチなし）。

## 4. 12パターンの合成関数

3軸から2軸を選ぶ 3C2 = 6 組み合わせ × 2合成モード = 12パターン。

### 4.1 加重和モード（軸が独立に寄与）

```
R_sum(α, β) = α · R_a + β · R_b
α + β = 1, α ≥ 0, β ≥ 0
```

### 4.2 積モード（一方の欠落が致命的）

```
R_prod(α, β) = R_a^α · R_b^β
α + β = 1, α ≥ 0, β ≥ 0
```

### 4.3 12パターン一覧

| # | 軸ペア | モード | 合成関数 R | 損失 L = 1 - R |
|---|--------|--------|-----------|----------------|
| 1 | struct × context | 加重和 | α·R_s + β·R_c | 1 - (α·R_s + β·R_c) |
| 2 | struct × context | 積 | R_s^α · R_c^β | 1 - R_s^α · R_c^β |
| 3 | struct × qualia | 加重和 | α·R_s + β·R_q | 1 - (α·R_s + β·R_q) |
| 4 | struct × qualia | 積 | R_s^α · R_q^β | 1 - R_s^α · R_q^β |
| 5 | context × qualia | 加重和 | α·R_c + β·R_q | 1 - (α·R_c + β·R_q) |
| 6 | context × qualia | 積 | R_c^α · R_q^β | 1 - R_c^α · R_q^β |
| 7 | struct (単独) | 加重和 | R_s | 1 - R_s |
| 8 | struct (単独) | 積 | R_s | 1 - R_s |
| 9 | context (単独) | 加重和 | R_c | 1 - R_c |
| 10 | context (単独) | 積 | R_c | 1 - R_c |
| 11 | qualia (単独) | 加重和 | R_q | 1 - R_q |
| 12 | qualia (単独) | 積 | R_q | 1 - R_q |

> **注**: 単独軸（#7-12）は加重和・積が退化して同一になる。形式的完全性のため12パターンとして列挙するが、実質的に異なるのは6パターン（#1-6）。

### 4.4 パターン選択の指針

| 翻訳ペア | 推奨パターン | 根拠 |
|---------|------------|------|
| 数学⇔形式言語 | #1 or #2 (struct×context) | R_qualia ≈ N/A |
| 音楽⇔創作 | #3 or #4 (struct×qualia) | 文脈より構造と質感の対応が支配的 |
| 創作⇔自然言語 | #5 or #6 (context×qualia) | 構造より感覚と前提共有が重要 |
| 科学論文→ニュース | #1 or #2 (struct×context) | 数式構造の保存 + 前提知識の保存が主軸 |

**加重和 vs 積の選択基準:**
- **加重和**: 一方の軸が低くても他方で補償可能な場合（例: 数式は失われたが文脈は完全に保持）
- **積**: 両軸が同時に必要な場合（例: 構造が0なら文脈があっても意味をなさない）

## 5. 逆算アルゴリズム

### 5.1 入力→出力

**入力:**
1. ソース要素集合 S（重要度 π 付き）
2. ターゲット要素集合 T
3. 対応関係 E（二部グラフの辺集合、対応強度 w 付き）
4. 各マッチペアの軸別距離 d_struct, d_context, d_qualia

**出力:**
- R_struct, R_context, R_qualia
- 選択されたパターンに基づく合成 R と損失 L

### 5.2 手続き

```
ALGORITHM InverseLoss(S, T, E, d_struct, d_context, d_qualia, π, pattern, α, β):

  # Step 1: 要素分類
  S_M ← {s ∈ S : ∃t, (s,t) ∈ E}
  S_U ← S \ S_M

  # Step 2: 軸別要素損失計算
  FOR each axis ∈ {struct, context, qualia}:
    FOR each sᵢ ∈ S:
      IF sᵢ ∈ S_M:
        ℓᵢ[axis] ← min_{(sᵢ,tⱼ)∈E} d_axis(sᵢ, tⱼ)
      ELSE:
        ℓᵢ[axis] ← 1.0

  # Step 3: 軸別復元度
  FOR each axis:
    R[axis] ← 1 - Σᵢ πᵢ · ℓᵢ[axis] / Σᵢ πᵢ

  # Step 4: パターン合成
  SWITCH pattern:
    CASE sum(a, b):  R_total ← α · R[a] + β · R[b]
    CASE prod(a, b): R_total ← R[a]^α · R[b]^β

  L_total ← 1 - R_total
  RETURN (R_struct, R_context, R_qualia, R_total, L_total)
```

## 6. 距離関数 d の具体化（Phase 1: 科学論文→ニュース）

科学論文→ニュース記事の翻訳ペアでの操作的定義:

### 6.1 d_struct: 構造距離

```
d_struct(s, t) = 1 - (命題構造の一致度)
```

**測定方法:**
- ソース要素 s を命題論理に還元: P(x) → Q(y) 等
- ターゲット要素 t も同様に還元
- 一致度 = (保存された関係数) / (元の関係数)

**具体例:**
- 論文: "∀ε>0, ∃δ>0 s.t. |x-a|<δ ⟹ |f(x)-L|<ε"
- 記事: "関数が特定の値に近づく"
- d_struct ≈ 0.85（全称・存在量化子、含意関係、不等式構造がすべて失われている）

### 6.2 d_context: 文脈距離

```
d_context(s, t) = 1 - (前提条件の再現率)
```

**測定方法:**
- ソース要素 s の前提条件リストを抽出（適用範囲、制約条件、先行研究への依存）
- ターゲット要素 t に言及されている前提条件を照合
- 再現率 = (言及された前提数) / (元の前提数)

### 6.3 d_qualia: 質感距離

```
d_qualia(s, t) = 1 - (トーン・インパクトの一致度)
```

**測定方法:**
- 感情極性（neutral/positive/negative）の一致
- 確信度表現の対応（"suggests" vs "proves" vs "breakthrough"）
- 科学論文→ニュースでは「誇張度」の逆数として近似可能

## 7. 理論的性質

### 7.1 境界条件

- **完全翻訳**: E が全射（全ソース要素がマッチ）かつ d = 0 ⟹ R = 1, L = 0
- **完全損失**: S_U = S（マッチなし）⟹ R = 0, L = 1
- **単調性**: 対応関係を追加 or 距離を縮小 ⟹ R は単調増加

### 7.2 加重和 vs 積の性質差

| 性質 | 加重和 | 積 |
|------|--------|-----|
| 一方が0のとき | R = β·R_b > 0 | R = 0 |
| 両方が1のとき | R = 1 | R = 1 |
| 感度 | 線形 | 低い方の軸に敏感 |
| 適用場面 | 補償的（一方で補える） | 必須的（両方必要） |

### 7.3 Gentner構造写像理論との関係

Gentner (1983) の構造写像理論では、アナロジーは「関係の保存」を最大化する写像。
HTLF の R_struct はこの写像の忠実度に対応する。
具体的には:
- 一次関係の保存 → d_struct の命題レベル一致
- 高次関係の保存 → d_struct の論理構造レベル一致
- 属性の非保存 → d_qualia での損失として計上

### 7.4 Shannon情報理論との関係

Shannon (1948) のチャネル容量 C は構造的情報の最大伝送率。
HTLF は意味的情報に拡張:
- R_struct ≈ 構文的チャネル容量の利用率
- R_context ≈ 語用論的情報の保存率（Shannonが明示的に除外した層）
- R_qualia ≈ 体験的情報の保存率（情報理論の射程外、IITの Φ が候補指標）

## 8. 制限と今後の課題

1. **d の主観性**: d_qualia は本質的に主観的。複数アノテータの一致度で信頼性を担保する必要がある
2. **重要度 π の設定**: ドメイン専門家の判断に依存。自動推定手法の検討が必要
3. **多対多対応**: 現在の定式化は1対多を許容するが、多対多の場合の集約方法を精緻化する余地がある
4. **非対称性の明示的モデル化**: A→B と B→A で損失が異なることは自明だが、その関係の定量化は未着手
5. **累積損失**: 多段翻訳（A→B→C）での損失累積モデルは今後の拡張課題
