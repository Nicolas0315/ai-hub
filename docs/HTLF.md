# Holographic Translation Loss Framework (HTLF)

> **Origin:** Youta Hilono × Shirokuma discussion, 2026-02-28
> **Status:** Theoretical framework — pre-implementation

## 概要

記号体系（レイヤー）間の翻訳で失われる情報を、ホログラフィック原理のアナロジーで定量化するフレームワーク。Katalaの信頼性評価（KS29B以降）に「翻訳忠実度」軸を追加するための理論的基盤。

## 背景思想

Youta Hilono の設計哲学に基づく：

- **心のモジュール性** → 認知は独立モジュールの集合体 → ソルバーも分離する
- **弱いサピア＝ウォーフ仮説** → 言語が思考を制約する → 言語圏ごとにバイアスが出る
- **中国語の部屋 + プラグマティズム** → 「理解しているか」より「理解として機能しているか」で評価する
- **オースティンの言語行為論** → 言葉は記述ではなく行為 → LLMの出力も行為として評価する

さらに分析哲学、パラダイム論、数理哲学、数学基礎論、論理学、非ユークリッド幾何学、物理学（ホログラフィック原理、相対論、量子論、M理論等）からの影響。

### 核心テーゼ

> 「①数学 ②形式言語 ③自然言語 ④音楽 ⑤創作——全部突き詰めると自然言語で、文脈依存の記号の交換では？」
> — Youta Hilono

## 5レイヤー

| # | レイヤー | 特徴 |
|---|---------|------|
| ① | 数学 | 文脈フリーに近い。抽象度最高。クオリアなし |
| ② | 形式言語（プログラム等） | 実行セマンティクスを持つ。数学に近いが計算機に束縛 |
| ③ | 自然言語 | 万能ハブだが浅い。文脈依存度最高 |
| ④ | 音楽 | 時間的・体験的質感が本質。形式化で最も失われるものが多い |
| ⑤ | 創作（視覚芸術等） | 音楽と隣接。翻訳者の解釈が不可避的に介入する |

## 3軸復元度モデル

### 定義

- **バルク** = 元のレイヤーにおける意味の全体（構造・文脈・質感すべて）
- **境界** = 翻訳先レイヤーでの表現（表層に現れた記号列）
- **復元可能性 R** = 境界からバルクをどこまで再構成できるか（0〜1）

### 3軸

- **R_struct**（構造復元度）: 関係性・演算・論理構造がどこまで戻せるか
- **R_context**（文脈復元度）: 翻訳先の記号列から元レイヤーの前提知識をどこまで復元できるか
- **R_qualia**（質感復元度）: 体験的・感覚的な意味がどこまで戻せるか

### 翻訳損失

```
L = 1 - R
```

## 12パターンの損失プロファイル

3軸から2軸を選ぶ組み合わせ: 3C2 = 6通り
各組み合わせに対して2つの合成モード:

```
加重和モード: R = Σ w_i × R_i   （軸が独立に寄与）
積モード:     R = Π R_i^(w_i)    （一つの欠落が致命的）
```

6組み合わせ × 2モード = **12パターン**

### 6つの軸組み合わせと支配的なレイヤーペア

| # | 組み合わせ | 支配的な翻訳ペア |
|---|-----------|-----------------|
| 1 | struct × context | 数学⇔形式言語（質感が存在しない） |
| 2 | struct × qualia | 音楽⇔創作（文脈より構造と質感の対応が鍵） |
| 3 | context × qualia | 創作⇔自然言語（構造より感覚と前提共有） |
| 4 | struct 単独加重 | 形式言語内の変換（トランスパイル等） |
| 5 | context 単独加重 | 自然言語間の翻訳（日→英等） |
| 6 | qualia 単独加重 | 音楽ジャンル間の翻訳（ジャズ→クラシック編曲等） |

**翻訳ペアごとに12パターンのどれに該当するかを自動分類する**のが実装上の最初の課題。

## レイヤー間翻訳マトリクス（定性的見積もり）

| 翻訳方向 | R_struct | R_context | R_qualia | R (総合) | L (損失) |
|---------|----------|-----------|----------|----------|----------|
| 数学→形式言語 | 0.95 | 0.80 | N/A | 0.88 | 0.12 |
| 数学→自然言語 | 0.70 | 0.50 | N/A | 0.60 | 0.40 |
| 音楽→自然言語 | 0.40 | 0.30 | 0.10 | 0.27 | 0.73 |
| 音楽→創作(視覚) | 0.30 | 0.40 | 0.50 | 0.40 | 0.60 |
| 形式言語→音楽 | 0.05 | 0.05 | 0.00 | 0.03 | 0.97 |
| 創作→自然言語 | 0.50 | 0.40 | 0.15 | 0.35 | 0.65 |

> 数値は定性的な見積もり。実装時に実験で較正する前提。

## 発見されたパターン

1. **隣接レイヤーは損失が小さい**: 数学⇔形式言語、音楽⇔創作
2. **遠いレイヤーほど翻訳不能領域が出る**: 形式言語→音楽はほぼ不可能（L=0.97）
3. **自然言語は万能ハブだが浅い**: どこからも翻訳可能だが常に解像度が落ちる。R_qualiaが特に犠牲になる
4. **体験的質感が最も失われやすい**: 音楽の緊張感、絵画の崇高さ
5. **非対称性**: A→BとB→Aで損失が異なる
6. **二段翻訳で損失が累積**: 自然言語ハブ経由は直接翻訳より損失が大きい

## Katalaへの実装パス

### Phase 1: 科学報道の翻訳忠実度
- 科学論文（数学/形式言語レイヤー）→ニュース記事（自然言語レイヤー）の翻訳損失Lを算出
- KS29Bの信頼性スコアに「翻訳忠実度」軸を追加
- **最初のユースケース**: 「この記事は元論文からどれだけ情報が失われているか」のスコア化

### Phase 2: プロファイル自動分類
- 翻訳ペアが12パターンのどれに該当するかを自動判定
- 各パターンに最適な重み付けを実験的に較正

### Phase 3: マルチレイヤー信頼性評価
- 同一の主張が複数レイヤーで表現されている場合、レイヤー間の整合性を評価
- 「数式と本文の説明が矛盾していないか」等の検出

## ホログラフィック原理との構造的類似

- ホログラフィック原理: d次元の境界情報が(d+1)次元のバルクを完全に記述する
- HTLF: 翻訳先の表層表現（境界）から元の意味の全体（バルク）をどこまで復元できるか
- 「境界の情報がバルクを記述する」≒「表層の記号から深層の意味を復元できるか」
- Katalaが取り組む「情報の信頼性」問題そのもの

## R_context Interpretation Selector（2026-03-01追加）

**Origin**: #dev-katala-law（Youta Hilono の洞察）

同一テキストに対するR_contextが時代・解釈体系によって異なる値を取る問題への対応。法律テキストで顕著だが、聖書解釈・憲法解釈・古典文学の再読など自然言語レイヤー全般に適用可能。

### メカニズム

```
テキスト T（不変）
  × 解釈体系 I_α（R_context = x_α）
  × 解釈体系 I_β（R_context = x_β）
  → current_context_selector: I_β（外部注入）
```

- R_contextの定義自体は不変
- 「どの解釈体系が現在有効か」を外部から選択する機構を追加
- 複数の解釈体系が並存する状態を自然にモデル化
- current_context_selectorのソース: 最新判例、法改正、学術的コンセンサス更新等

### レイヤー間翻訳マトリクスへの影響

法律テキストの翻訳では、R_contextが翻訳段階ごとに**増減する**（判例蓄積で文脈が事後充填される）。これは音楽・コードでは発生しないパターン。Interpretation Selectorにより、この現象を「異なる解釈体系のR_context比較」として統一的に扱える。
