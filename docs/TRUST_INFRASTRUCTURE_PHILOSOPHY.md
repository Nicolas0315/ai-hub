## 1. なぜ「情報の信頼性」が根本問題なのか

### 1.1 データ収集→権力行使の歴史的連鎖

歴史は繰り返し、**情報の収集・独占・工作が支配の道具になる**ことを示してきた。

**ナチス政権（1930年代〜）:**
- ユダヤ人の人口統計・職業・居住地を系統的にデータ化
- IBMのパンチカード技術が収容所管理に転用（Edwin Black "IBM and the Holocaust", 2001）
- データ収集が大量虐殺の実務的インフラになった

**東ドイツ・シュタージ（1950〜1990年）:**
- 人口1,600万人に対して非公式協力者18万人
- 精緻な市民監視・ファイル化による社会統制
- 「知ること＝支配すること」の極限形態

**結論:** EUがGDPRで「プライバシーは人権」と定めたのは抽象的理念ではなく、**具体的な歴史的暴力への応答**である。ルールの正当性は歴史的文脈から生まれる。

**Katalaへの示唆:**  
情報信頼性の基準を設計する際、「なぜそのルールが必要か」という歴史的文脈を持たせることで、単なる技術仕様を超えた正当性が生まれる。

## 2. インフラレベルの工作問題

### 2.1 オペレーション・ルビコン（1945〜2018年）

CIAとドイツBNDがスイスの暗号機メーカー「クリプトAG」を秘密裏に所有し、100カ国以上に意図的に脆弱化した暗号機を販売。購入国は安全な通信を行っていると信じていた。

**核心的教訓:**
> **インフラレベルで工作された場合、ユーザーレベルのセキュリティ対策は原理的に無意味。**

「中立な第三者のはずのスイス企業が、実は最大の当事者だった」——これが70年間機能した理由。

### 2.2 現代の類似事例

| 事例 | 対象 | 確認度 |
|---|---|---|
| NSA TAO部隊によるCiscoルーター輸送途中改ざん | ネットワーク機器 | ✅ スノーデン文書で確認 |
| Supermicroマザーボードへの米粒大チップ挿入 | サーバーハードウェア | 🟡 Bloomberg報道・否定あり |
| Dual EC_DRBG乱数生成器バックドア | 暗号ソフトウェア | ✅ NSA関与確認 |
| Juniper Networks VPN製品への二重侵入 | ネットワーク機器 | ✅ 確認済み |

### 2.3 Katalaへの示唆

**スコアリング主体そのものの独立性問題:**

TrustScorer / ConsensusEngineが機能するためには、スコアリングを行う主体が「クリプトAGになっていないか」を継続的に検証できる仕組みが必要。

具体的に問われること：
- スコアリングアルゴリズムはオープンソースか（監査可能性）
- スコアに影響を与える主体（資金提供者・運営者）は誰か（利益相反の開示）
- スコアリング基準の変更は検証可能なログを残すか（改ざん不可能性）

> **「誰がスコアリングしているか」の検証可能性がない限り、どんなに優れたアルゴリズムもクリプトAGと同じ構造的脆弱性を持つ。**

## 3. 科学的プロトコルの反証可能性と情報スコアリング

### 3.1 科学的知識の本質的な位置づけ

科学的プロトコル自体が「反証可能な仮説の集合」である——これはクーン・ポパー以降の科学哲学の核心。

重要な区別：
- **「科学は信頼できない」** → 誤り
- **「科学的知識は反証可能で更新される」** → 正しい認識

この区別はKatalaの設計に直結する。

### 3.2 Katalaのスコアリング設計への応用

**現在の表現（避けるべき）:**
> 「この情報は信頼できる」

**推奨される表現:**
> 「現時点で利用可能な証拠と検証プロセスに基づき、この情報の信頼度スコアは X/100 である。スコアは新たな証拠によって更新される」

これは単なる言葉の問題ではなく、**設計思想の問題**：
- 静的な「真/偽」判定 → 権威主義的・工作されやすい
- 動的な「現時点での支持度」 → 科学的・更新可能・透明

### 3.3 観測の理論負荷性

あらゆる観測は理論的前提のフィルターを通している（Hanson）。Katalaが評価する「情報」も、評価基準というフィルターを通して解釈される。

**設計上の帰結:**
- 評価基準を複数公開し選択可能にする
- 評価基準間の差異を可視化する
- 「どの基準で評価したか」をスコアに添付する

## 4. 認識論的誠実さ：「真実プラットフォーム」の罠

### 4.1 「真実」を宣言するプラットフォームの問題

歴史的に「真実を守る」と宣言した権威は、自らが工作の主体になりやすい。

**構造的問題:**
- 真実の定義権＝権力
- 「中立な審判者」の外観を維持しながら特定の利益を優先（クリプトAGと同じ）

### 4.2 Katalaが目指すべき位置

Katalaは「真実を決める」プラットフォームではなく、**「信頼性評価のプロセスを透明化する」インフラ**として設計されるべき。

| 避けるべき設計 | 目指す設計 |
|---|---|
| 「この情報は真実」と宣言 | 「このプロセスでこのスコアが算出された」を開示 |
| 中央集権的な評価主体 | 分散・競合する評価主体の共存 |
| 静的なルール | 反証可能で更新可能な基準 |
| ブラックボックスアルゴリズム | オープンソース・監査可能 |

## 5. 第三者機関の設計原則

### 5.1 「本当の第三者」の条件

オペレーション・ルビコンが示した教訓：第三者性は外観ではなく構造で担保されなければならない。

**第三者機関の必要条件:**
1. **資金の独立性** — スコア対象から資金提供を受けない
2. **アルゴリズムの公開** — 評価プロセスが検証可能
3. **意思決定のログ** — 基準変更・判断の履歴が改ざん不可能な形で保存
4. **利益相反の開示** — 評価者と評価対象の関係の透明性
5. **競合の許容** — 単一の評価機関への依存を避ける設計

### 5.2 ブロックチェーンの役割と限界

改ざん不可能なログという点でブロックチェーン技術は有効だが、**ログの内容の正しさはブロックチェーンが保証しない**。

「正直に記録された嘘」は検証できない——記録の形式と内容の信頼性は別問題。
