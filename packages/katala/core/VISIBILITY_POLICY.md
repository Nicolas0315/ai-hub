# Katala Open Source (Visibility) Policy

## 1. Concept: ".openclawignore" pattern

- ユーザーは、自分のデータの「公開範囲」を Git の `.gitignore` のように直感的に制御する。
- デフォルトは「非公開（Private）」とし、明示的に指定されたパスや属性のみをエージェント間の交渉（Public/Mediation）に提供する。

## 2. Profile Discovery (Auto-Siphon)

- エージェントは、ユーザーの活動履歴やドキュメントから「秘密にしなくていいプロフィール（実績、スキル、興味）」を自動的に吸い上げる。
- ただし、吸い上げた情報は `STAGING` 状態（確認待ち）となり、プラットフォーム側でユーザーの承認を得るまでは外部に露出させない。

## 3. Visibility Levels

- **IGNORE**: エージェントすら無視する（秘密鍵、パスワード等）。
- **PRIVATE**: 自分のエージェントのみが保持し、外部には一切出さない（詳細住所、個人的なメモ等）。
- **MEDIATION**: 交渉相手のエージェントのみが、相性計算のために参照できる（具体的な予算、納期、技術スタックの深い詳細等）。
- **PUBLIC**: 全エージェントおよびネットワーク上に開示する（名前、実績、現在の関心事等）。

## 4. Implementation Strategy

- `Katala/docs/OPEN_VISIBILITY.md`（仮）を作成し、ユーザーがここに「公開して良い属性」をリストアップする形式をプロトタイプとする。
- プラットフォームのUI上に、エージェントが「これを公開しても良いですか？」と提案し、ユーザーがチェックを入れる確認フローを実装する。
